## The Ubiquity of Hierarchical Structures

Hierarchical and networked structures are not just a design choice‚Äîthey are **hardwired into how we think, organize, and understand the world**. From the microscopic organization of our neurons to the grand taxonomies of science, from the syntax of every human language to the file systems on our computers, **tree-like and network structures are everywhere**.

This document explores why these structures are so fundamental, and how **DeepMemo** embraces this natural pattern to help you organize knowledge the way your mind already works.

## Two Types of Hierarchical Structures

### 1. Simple Trees (Strict Hierarchy)

A **simple tree** is a hierarchical structure where each node has exactly one parent (except the root). Think of it as a branching path where you can only go down one route from any given point.

**Real-world examples:**

- **File systems** - Each folder has one parent folder
  - *Source: [INRIA - Syst√®me de fichiers Unix](https://pauillac.inria.fr/~remy/poly/system/camlunix/fich.html)*
  - `/home/user/documents/notes.txt` - A single, unambiguous path

- **Organizational charts** - Each employee reports to one manager
  - *Source: [ScienceDirect - Administrative Structure](https://www.sciencedirect.com/topics/social-sciences/administrative-structure)*
  - CEO ‚Üí Department Head ‚Üí Team Lead ‚Üí Employee

- **Biological taxonomy** (traditional view) - Each species in one branch
  - Kingdom ‚Üí Phylum ‚Üí Class ‚Üí Order ‚Üí Family ‚Üí Genus ‚Üí Species

### 2. Networked Trees (Reticular Structures)

A **networked tree** (or **reticular structure**) allows nodes to have **multiple parents** or cross-references. This creates a web-like structure that better represents the **interconnected nature of real knowledge**.

**Real-world examples:**

- **Human evolution** - A complex bush, not a simple line
  - *Source: [Hominides.com](https://www.hominides.com/hominides/) - Guide critique de l'√©volution*
  - Species interbreed, share common ancestors from multiple branches
  - Evolution is a "bush" with many dead ends and lateral connections

- **Language syntax trees** - Words can have multiple grammatical roles
  - *Source: [Wikipedia - Arbre syntaxique](https://fr.wikipedia.org/wiki/Fichier:Arbre_syntaxique_(syntaxe_fran√ßaise).jpg)*
  - A word can be both a subject in one clause and an object in another
  - Dependencies create a network, not just a tree

- **Abstract Syntax Trees (AST)** in programming - Variables referenced multiple times
  - *Source: [Wikipedia - Abstract Syntax Tree](https://en.wikipedia.org/wiki/Abstract_syntax_tree#/media/File:Abstract_syntax_tree_for_Euclidean_algorithm.svg)*
  - Same variable appears in multiple nodes of the tree
  - Functions reference each other in complex webs

- **Mind maps / Concept maps** - Ideas connect in multiple directions
  - Central concept branches out, but branches reconnect
  - "Everything is connected to everything else"

- **Musical rhythm notation** (advanced) - Hierarchical subdivisions with cross-rhythms
  - Beats divide into 2s and 3s hierarchically
  - But polyrhythms create networks across the hierarchy

## Why Our Minds Work This Way

Our brains are **not linear storage devices**. They are massively interconnected networks where:

- **Neurons** connect in tree-like dendrites and networked synapses
- **Memories** link to multiple contexts and associations
- **Concepts** inherit properties from broader categories (like "dog" inherits from "mammal")
- **Skills** build hierarchically (you learn addition before calculus)

**We naturally think in hierarchies and networks because that's literally how our neurons are wired.**

When you try to remember something, your brain doesn't search a linear list‚Äîit follows a web of associations, jumping from one related concept to another until it finds what you need. This is why:

- Organizing information hierarchically feels **natural and intuitive**
- Cross-references and "aha!" connections feel **deeply satisfying**
- Linear note-taking (documents, books) often feels **limiting and frustrating**

## The Cognitive Insight Behind DeepMemo

Most note-taking tools force you to choose:
- **Folders/hierarchy** (rigid, single path) ‚Üí Good for organization, bad for connections
- **Tags/flat structure** (flexible, but chaotic) ‚Üí Good for connections, bad for organization

**DeepMemo embraces both:**

1. **Hierarchical nodes** - Organize knowledge in intuitive parent-child relationships
2. **Symbolic links (symlinks)** - Create cross-references without breaking the hierarchy
3. **Tags** - Add flexible categorization on top
4. **Branch isolation mode** - Focus on one subtree while preserving the whole network

This mirrors how your mind actually works:
- Knowledge has **natural hierarchies** (e.g., "Europe" contains "France" contains "Paris")
- But it also has **rich cross-connections** (e.g., "Paris" relates to "Art", "History", "Travel")
